# AI 心脏健康小程序（cardiac_risk_miniapp）

一个微信小程序：以智能穿戴设备 + 手动录入的心脏监测数据为基础，用 Apple Watch 风格展示指标，结合大模型 AI 给出风险评估、长短期建议与趋势分析，并提供紧急联系人 / 一键急救。

> 本地运行 / 课堂作业版本。数据存在手机本地（`wx.storage`），首次启动从 `mockdata/seed.js` 播种演示数据。AI 走大模型 API（前端直连，仅本地用）。
> 详细需求见 [需求文档.md](需求文档.md)，当前实现 **M1 + M2 + 部分 M3**（AI 问答、就医建议分级）。

---

## 功能

底部四个 Tab：

- **首页**：今日心脏概览（渐变 Hero + 静息心率大数字 + 风险等级）、心脏指标网格（心率/HRV/血压/血氧/脉搏，按阈值显示状态色）、生活方式指标、**短期建议（今天可以做）**、**AI 风险评估**（风险环 + 影响因素 + 长期管理）、**就医建议分级**（可观察 / 建议咨询 / 尽快就医）。
- **趋势**：可切换指标的**趋势折线图**（canvas 绘制）+ 均值/最高/最低、**历史记录列表**、一键生成**长期 AI 趋势分析**。
- **问答**：基于你的档案与最近数据的 **AI 多轮对话**，可追问心率/血压/睡眠等问题。
- **我的**：可编辑的**健康档案**（基础信息 + 健康背景，作为 AI 上下文）、**紧急求助**（一键拨 120 + 默认联系人快拨）、入口跳转录入与联系人管理。

二级页面：

- **录入体征**：手动新增一条记录（心率/血压/血氧/HRV 等），保存后纳入仪表盘与评估。
- **紧急联系人**：增删、设默认、一键拨打 + 顶部一键拨 120。

## 运行前准备（两步）

### 1. 配置大模型 API
打开 [`config.js`](config.js) 填入 key（OpenAI 兼容格式，换厂商只改三项，文件内有各家地址注释）：

```js
module.exports = {
  baseUrl: 'https://api.deepseek.com/chat/completions',
  apiKey: 'sk-你的API_KEY',
  model: 'deepseek-chat',
}
```

> ⚠️ key 写在前端只适合本地测试。正式上线需由后端 / 微信云函数中转，否则会泄露。

### 2. 关闭域名校验（本地必需）
微信开发者工具：**详情 → 本地设置 → 勾选「不校验合法域名、web-view、TLS 版本以及 HTTPS 证书」**，否则 `wx.request` 会被拦截。

完成后点「编译」即可。AI 评估与趋势分析需要网络与有效 key；指标仪表盘、录入、联系人即使没 key 也能用。

## 数据格式

体征记录（`store.addRecord` / seed）：

```js
{
  id, 来源: '设备'|'手动', device, date: '2026-05-22', time: '08:00',
  metrics: { 静息心率, 平均心率, 心率变异性HRV, 血氧饱和度, 收缩压, 舒张压, 脉搏, 睡眠时长, 日步数, ... },
  异常事件: ['检测到 1 次疑似房颤']
}
```

个人档案：`{ 性别, 年龄, 身高, 体重, 高血压, 糖尿病, 吸烟, 家族史, 用药:[...] }`
紧急联系人：`{ id, 姓名, 关系, 电话, 是否默认 }`

- **改阈值/颜色**：[`utils/metrics.js`](utils/metrics.js) 的 `band(...)` 与 `accent`。
- **改演示数据**：[`mockdata/seed.js`](mockdata/seed.js)（仅首启播种一次；清缓存后重新播种）。
- **接真实设备**：把设备数据整理成上面的 record 结构，调用 `store.addRecord` 即可。

## 设计说明

参考 Anthropic `frontend-design` 原则：统一主色（深玫红）+ 鲜明指标色、强字号字重层次、渐变 Hero 与柔和层叠阴影、入场错落淡入动效。设计令牌集中在 [`app.wxss`](app.wxss)（CSS 变量）。

> ⚠️ WXSS 选择器不能含中文，状态/等级类一律用英文（high/mid/low、good/warn/bad、up/down/flat）。
