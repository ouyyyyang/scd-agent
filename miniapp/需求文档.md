# AI 心脏健康小程序 · 需求分析文档

> 版本 v0.1 · 2026-05-23 · 状态：待评审
> 目标：从「读一份数据 + AI 评估」的雏形，扩展为一个**完整的 AI 心脏健康管理小程序**。
> 本文档先做需求分析，确认后再据此修改代码。

---

## 1. 项目背景与定位

- **是什么**：一款面向个人的心脏（心血管）健康管理小程序。通过穿戴设备数据 + 用户录入，结合大模型 AI，给出风险评估、长短期趋势分析、个性化建议，并在异常时联动紧急联系人。
- **服务对象**：用户本人（单用户视角，无后台管理多人）。
- **不是什么**：不是医疗器械，不做诊断，所有 AI 输出仅供参考、不替代医生。
- **当前阶段**：课堂作业 / 本地运行；数据来自项目内 `mockdata/`，AI 走大模型 API（前端直连，仅本地用）。

---

## 2. 目标用户与画像

| 画像 | 特征 | 核心诉求 |
| --- | --- | --- |
| **健康关注者**（年轻人） | 有手环/手表，关注心率、睡眠、运动 | 看趋势、获得作息/运动建议、保持习惯 |
| **慢病/高危人群**（中老年） | 高血压、心率异常、有家族史 | 风险预警、用药/复诊提醒、异常时能求助 |
| **康复/术后人群** | 需长期监测 | 长期数据追踪、向医生导出报告 |

共同诉求：**看得懂自己的心脏状况**、**知道该做什么**、**出事时有人知道**。

---

## 3. 典型使用场景

1. **每日查看**：早上打开小程序，看昨夜睡眠、静息心率、今日整体状态与一句话 AI 点评。
2. **趋势回顾**：周末看近 7 天/30 天的心率、HRV、血压趋势，了解变化方向。
3. **异常预警**：设备捕捉到疑似房颤/夜间血氧偏低，小程序给出提示并建议就医。
4. **主动录入**：在家用血压计测了血压，手动录入，纳入分析。
5. **紧急求助**：感觉不适，一键拨打紧急联系人 / 急救电话，并可附带最近数据。
6. **就医准备**：导出一份近期数据报告，看病时给医生参考。

## 4. 功能需求

按模块列出。优先级：**P0=MVP 必做**，**P1=重要**，**P2=可后续**。

### 4.1 数据采集与录入
- [P0] 读取穿戴设备数据（当前用 `mockdata/` 模拟，预留真实设备/导入接口）。
- [P0] **手动录入体征**：血压（收缩压/舒张压）、心率、体温、体重等，带时间戳。
- [P1] 多次录入形成时间序列（支持一天多条）。
- [P2] 蓝牙/健康数据导入（微信运动、设备 SDK）——平台能力受限，列为远期。

### 4.2 实时 / 短期指标（仪表盘）
- [P0] Apple Watch 风格指标卡：静息心率、平均心率、HRV、血氧、睡眠、步数等。
- [P0] 每张卡按阈值显示状态色（正常/关注/异常）。
- [P1] 顶部"今日概览"大圆环 + 最近同步时间。
- [P1] **短期 AI 点评**：针对"今天/最近一次"数据的一句话提醒与即时建议。

### 4.3 长期趋势与历史
- [P0] **历史数据**页：按日/周/月查看记录列表。
- [P1] **趋势图表**：心率、HRV、血压、睡眠的折线/柱状趋势（周/月/年）。
- [P1] **长期 AI 分析**：基于一段时间数据，总结趋势（变好/变差）、给出长期管理建议。
- [P2] 与个人目标对比（如静息心率目标、睡眠目标达成率）。

### 4.4 AI 健康建议
- [P0] **风险评估**：风险等级 + 整体风险百分比 + 主要影响因素 + 建议（现有）。
- [P0] **建议分层**：
  - *短期/即时*：今天可做的（如"今晚早睡""减少咖啡因"）。
  - *长期/管理*：持续性的（如"控制血压在 130/80 以下""每周 150 分钟有氧"）。
- [P1] **AI 问答**：用户就自己的数据向 AI 提问（"我最近心率为什么偏高？"）。
- [P2] 个性化：结合个人档案（年龄、病史、用药）让建议更准。

### 4.5 异常预警与提醒
- [P1] 异常事件展示（疑似房颤、夜间血氧低、心率骤升等）。
- [P1] **就医建议分级**：提示"可观察 / 建议咨询 / 尽快就医"。
- [P2] 用药提醒、复诊提醒、测量提醒（本地定时/订阅消息）。

### 4.6 紧急联系人 ⭐（用户明确要求）
- [P0] **管理紧急联系人**：增删改，含姓名、关系、电话。
- [P0] **一键拨号**：`wx.makePhoneCall` 直接拨打联系人/急救电话（120）。
- [P1] **求助时附带信息**：拨号前展示/复制最近关键数据，便于告知。
- [P2] 紧急情况自动分享位置 / 一键发送预设短信。

### 4.7 个人档案
- [P1] 基础信息：性别、年龄、身高体重。
- [P1] 健康背景：高血压/糖尿病、吸烟、家族史、当前用药。
- [P1] 这些信息作为 AI 评估的上下文，提升准确度。

### 4.8 报告与分享
- [P2] 生成阶段性健康报告（图文），可保存图片 / 分享给医生或家人。

### 4.9 设置
- [P1] 指标阈值/单位、提醒开关。
- [P1] 大模型配置（已在 `config.js`）、免责声明、关于。

## 5. 信息架构（页面结构）

建议底部 **TabBar 三/四个入口**：

```
TabBar
├── 首页（今日概览）      指标仪表盘 + 短期 AI 点评 + 异常提示
├── 趋势（历史与分析）    历史列表 + 趋势图表 + 长期 AI 分析
├── 我的（档案与设置）    个人档案 / 紧急联系人 / 设置
└──（可选）AI 问答        与 AI 就自身数据对话

二级页面
├── 录入体征数据
├── 紧急联系人管理 / 编辑
├── 风险评估详情
├── 历史详情（某条记录）
└── 健康报告
```

> 注：当前代码是单页（首页=仪表盘+评估）。落地时按上面拆分，**首页保留仪表盘并下沉风险评估到详情**，新增趋势页、我的页、录入页、紧急联系人页。

## 6. 数据模型（本地存储 / 未来云端）

```js
// 用户档案 profile
{ 性别, 年龄, 身高, 体重, 高血压, 糖尿病, 吸烟, 家族史, 用药:[...] }

// 体征记录 record（按时间序列存多条）
{
  id, 来源: '设备'|'手动', date, time,
  metrics: { 静息心率, 平均心率, HRV, 血氧, 收缩压, 舒张压, 体温, 睡眠时长, 步数, ... },
  异常事件: [ ... ]
}

// 紧急联系人 contact
{ id, 姓名, 关系, 电话, 是否默认 }

// AI 结果（可缓存，避免重复请求）
{ recordId, riskLevel, riskScorePercent, summary, factors:[], shortTermTips:[], longTermTips:[] }
```

存储方案：
- **MVP**：`wx.setStorageSync` 本地持久化（够课堂作业用）。
- **后续**：微信云开发（云数据库）或自建后端，支持多设备同步。

## 7. AI 能力设计

- **输入**：用户档案 + 一条或一段时间的体征数据。
- **输出（结构化 JSON）**：在现有 `riskLevel / riskScorePercent / summary / factors / suggestions` 基础上，建议拆分建议为：
  ```json
  { "shortTermTips": ["今天/本周可做的"], "longTermTips": ["长期管理目标"] }
  ```
- **两类调用**：
  1. *单次评估*：基于最近一条数据 →（现有逻辑增强）。
  2. *趋势分析*：基于一段序列 → 总结变化趋势与长期建议。
- **AI 问答**（P1）：多轮对话，system prompt 注入用户数据上下文。
- **安全**：key 仅本地用；上线需后端/云函数中转。所有输出附免责声明。

---

## 8. 非功能需求

- **隐私合规**：健康数据敏感；明确告知数据用途、本地存储、可清除；上线需隐私协议。
- **医疗免责**：全程标注"仅供参考，不替代医生诊断"；不出现确诊式表述。
- **性能**：AI 请求有 loading 与超时处理；趋势图数据量大时分页/采样。
- **可用性**：弱网/无 key/接口失败要有清晰提示与重试。
- **可维护**：指标阈值、AI prompt、数据样本均集中、易改（现已如此）。

---

## 9. 技术架构与平台约束（微信小程序）

- `wx.request` 需域名白名单，本地测试需关闭域名校验；上线需在后台配置 request 合法域名。
- **不能读任意本地磁盘文件**，模拟数据须打包进项目（已用 `mockdata/`）。
- 一键拨号用 `wx.makePhoneCall`；定时提醒用订阅消息（需用户授权，能力有限）。
- 图表可用 `canvas` 或第三方（如 ec-canvas / 自绘）。
- 前端直连大模型仅限本地；正式环境用**云函数**中转保护 key、并绕过域名限制。

## 11. 已确认的决策（2026-05-23）

1. **范围**：一次做到 **M1 + M2**（仪表盘+短期点评、手动录入、紧急联系人+拨号、本地存储、AI 建议分长短期、历史列表+趋势图、长期 AI 趋势分析、个人档案）。

4. **指标范围**：**以心脏指标为主**（静息/平均心率、HRV、血压、血氧、脉搏），睡眠/步数/久坐等作为辅助指标，排序靠后。
5. **趋势图**：纳入 M2，本轮实现。
6. **AI 建议**：拆 **短期(shortTermTips) + 长期(longTermTips)** 两栏；AI 问答留到 M3。

> AI 问答、异常预警分级、提醒、报告导出、真实设备接入、云端同步 → 留待 M3/M4。
